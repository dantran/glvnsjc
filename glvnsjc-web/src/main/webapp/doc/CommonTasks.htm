GLVNSJC DATABASE SYSTEM (GLVNSJCDB). 

 

Concept

GLVNSJCDB provides a WEB student/teacher tracking system for all GLVN schools.  Each school manages its own student database.

Scenarios

 

The following contains a list of common tasks happen during a typical school year.  A solution of each task shows how we can use GLDNSJCDB to implement the task.

Before first class, Frere sends out a preliminary list of new school year students.

 

The list is in a form of Microsoft Excel spreadsheet, which contains all students ‘record for the new school year.

 

Solution: GLVNSJCDB has an import feature, which can transfer student records from the spreadsheet into its database. Accounts with “admin” privilege can perform this task. 

 

School Principal divides classes into subclasses

 

For large size classes, a principal can divide a class into subclasses (i.e. VN 3A, VN3B, etc.)

 

Solution:  A principal can use a “principal” privilege account to search for all students of a particular Giáo Lý or Việ̣t Ngữ class and assign subclass (A, B, C, and D) to each student. A principal can designate a teacher to do this task as well.

 

An admin privilege account can create principal privileged account for each school.  A principal account can create teacher accounts for each school

 

Note: principal can only see student records of his/her own School

                 

Teacher would like to see student records

      

Solution: School principal creates "teacher" account for each teacher/teacher aid that is RESPONSIBLE to update student records.

                

Teacher uses this “teacher” account to search for student records of any particular class using a generic search form.

 

Teacher can lookup student using wild card search using name, phone, date of birth, class name, etc.

Frere sends a new update list of students.

 

After school start, Frere still receive new applications and update his database.  Frere will send a new spreadsheet with all students in a school year.

 

Solution: Use the import tool again to merge new spreadsheet to database.  The import tool will make sure not to overwrite previous changes made by principal/teacher

 

School begins and student class changes

 

After school starts, each school has it own way to reassign student class. For example:  assigning new VN class based on student’s Việt Ngữ level.

 

Solution:  Teacher of both VN and GL classes make changes accordingly to the actual number of students in their class. This task consists of:

 

·        For all students NOT in the class roster, the teacher will use the search engine to find the student based on name, phone, or date of birth and move those students to his/her own class.

 

·        For all students in the class roster but not present in the class, change the class assignment to none.

 

These tasks should be completed by the 4th weeks of school year so that the actual students and database are synchronized.

 

Transfer student to another school

This is the case where parent asks principal to allow his/her child to transfer from another school. Or student for some reason ends up in one school database and physically goes to a different school

 

Solution: A principal account can search for a student using the student transfer feature, which can search all schools‘ database, and move the student to his/her own school.

Grade

At the end school year, Each Giáo Lý, Việt Ngữ student receives a grade which can be Pass (P), (F), 1, 2, 3, and 4.

 

Solution: GLVNSJCDB provides a custom screen the assign grades to all students of a teacher’s class. Teacher assigns all grades in one screen before submission.

 

Principal can use certificate report to print certificates at the end of school year (no more last minute hand written award certificate)

School year end’s and student application labels:

At the end of each school year, each student receives school application for next year.  Each application has a label with student’s identification, birth date, and next year Giáo Lý Việt Ngữ class assignments.

 

Solution: Principal can use student application report to print student labels

 

ACCOUNT SUMMARY

 

Teacher: Can update student records in a school of a current school year.

 

Principal:  is a teacher but can create teacher account and transfer student to his/her own school

 

Master:  has principal privilege of all schools, can create principal/teacher account, and can add and delete student records

 

Admin: has the master privilege and can import Frere’s spreadsheet to database

 

 

TECHNOLOGY SUMMARY

 

GLVNSJCDB is developed under Tomcat servlet /JSP container, Struts model view controller 2 framework, Hibernate Object Relational Mapping Framework, Hypersonic SQL database
